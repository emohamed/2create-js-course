Конзолата на browser ви позволява да дебъгвате javascript код. Shortcut-а за нея е F12.

Изпълнение на javascript

В един HTML документ може да имате множество javascript тагове.

<script> таговете могат да съдържат src атрибут, който указва локация на javascript файл. В този случай, browser-а сваля този файл и го изпълнява в контекста на документа, където е извикан.

Преглед на javascript

  • Един от най-популярните езици за програмиране днес
  • Използван както за front-end(browser) така и за back-end(server) development
  • C-like синтаксис
  • Фунцкионален и обектно ориентиран
  • Whitespace-а почти винаги нямат значение
  • Редовете няма нужда да завършват с ;
  • Динамично типизиран език, променливите се декларират с var
  • Коментарите се игнорират; многоредовите коментари са между /* и */. Едноредовите започват с //

Типове от данни

В javascript има 2 вида данни: прости(basic types) и комплексни(complex types). Простите(понякога се наричат и скаларни) се състоят от една стойност. Всички комплексни типове са обекти и могат съдържат повече от 1 стойност.

Днес започваме с разглеждането на простите типове:

  • Numbers(числа)
  • Strings(низове)
  • Booleans(булеви)
  • Undefined и null

Numbers - Числа

Всички числа в javascript се третират като реални; например 5 вътрешно е 5.0.

Функции свързани с числа има в обекта Math: abs, ceil, max, min, random и други.

Деление на нула в javascript дава резултат безкрайност:

5 / 0 => Infinity

Ако операцията от числа не може да бъде извършена се получава стойност NaN(not a number):

5 / "Banana" => NaN

Numbers - oprerators

Оператори:

  • Събиране 5 + 6 = 11
  • Изваждане 5 - 6 = -1
  • Умножение 5 * 6 = 30
  • Деление 5 / 6 = 0.833...
  • Остатък от деление 5 % 6 = 1
  • Инкрементиране a++, a += b, ++a
  • Декрементиране a--, a -= b, --a
  • Обръщане на знак: -a => a *= -1
  • Форсиране на тип число: +a => a

Strings - Низове

String(низ) в javascript е текст обграден в двойни или единични кавички.

Низовете са обекти; промяната на низ създава нов низ.

Няма отделен тип character; той е низ с един елемент.

Обратната наклонена черта е escape character и се използва за да поставят специални символи в низа:

\n, \r, \t - whitespace

\", \' - quoted кавички

Низа не може да съдържа нов ред директно, трябва да се използва \n, \r или и двете

Concatenation("слепване") на 2 низа се прави с оператора плюс:

var name = "Emo";
var greeting = "Good evening";
console.log(greeting + ", " + name);
// => Good evening, Emo

Strings

var text = "Lorem ipsum dolor sit amet, consectetur adipisicing\r\n" + 
"elit. Corporis, ratione, accusantium fugit consectetur pariatur";

console.log("Message for " + text);

Разликата между \r и \n.

Strings - Низове (методи)

Всеки низ е обект и притежава properties и methods:

var s = "javascript strings";
console.log(s.length); 
// => 18

Методите са функции, които работят в контекста на низа. Те не променят низа, а връщат нов низ. Няколко примера:

  • s.indexOf()
  • s.lastIndexOf()
  • s.replace()
  • s.charAt()
  • s.slice()
  • s.substr()
  • s.toUpperCase(), .toLowerCase()

Booleans - булеви променливи

Booleans(булев тип) е тип съдържащ само 2 стойности: истина(true) и лъжа(false).

Всички стойности в останалите типове могат да се сведат до булева стойност.

Следните стойности се свеждат до false(falsy са):

  • false
  • null, undefined
  • ''(empty string)
  • 0, NaN

Всичко останало се свежда до true(truthy е).

Функции

Функциите са широка тема, която ще обсъдим в дълбочина по-късно. За момента само ще видим какъв е основния им синтаксис и какво представляват.

// Returns a numeric value between -1 and 1 that
// represents the sine of the angle given in degrees.
function sin_from_degrees(angle_degrees) {
    var angle_radians = (angle_degrees * Math.PI) / 180;
    var result = Math.sin(angle_radians);
    return result;
}
var angle_degrees = 90;
var res = sin_from_degrees(angle_degrees);
console.log("Sine of " + angle_degrees + " is " + res);
  • function - ключова дума дефинираща функция
  • comment - добре е функцията да има коментар оказващ какво е предназначението й
  • sin - име на функцията
  • angle_degrees - параметри на функцията
  • angle_radians - локални променливи на функцията
  • return - върната стойност

Функции: връщане на стойност

Функциите могат да върнат или да не върнат стойност. Когато не се връща стойност, функцията обикновенно има странични ефекти(side effects):

function log_matrix(message, matrix) {
    console.log(message);
    console.table(matrix);
}
log_matrix("Ping ping results", [
    ["", "Emo", "Stefan"],
    ["Game 1", "1:1", "2:1"],
    ["Game 2", "0:1", "0:2"]
]);

Резултата от функцията е undefined, но тя извършва полезна работа сама по себе си вместо да връща резултат.

Функции: преизползване

Целта на функциите да намалят сложността на кода и да го направят по лесно разбираем. Те ви дават възможност да изолирате определена операция, да й дадете име и да я викате множество пъти без да има нужда да я повтаряте: 

// Transforms degrees in radians
function degrees_to_radians(angle_degrees) {
	return (angle_degrees * Math.PI) / 180
}
// Returns cosine from an angle in degrees
function cos_from_degrees(angle_degrees) {
	return Math.cos(degrees_to_radians(angle_degrees));
}
// Returns sine from an angle in degrees
function sin_from_degrees(angle_degrees) {
	return Math.sin(degrees_to_radians(angle_degrees));
}
console.log(sin_from_degrees(0));
console.log(cos_from_degrees(0));

Undefined and Null

В javascript има 2 вида празни променливи:

  • undefined
  • null

Техническата разлика между тях е тънка и извън обхвата на това обучение.

Type Conversion: промяна на тип

Когато оператор трябва да работи с различни типове, javascript вътрешно прави обръщане на типовете за да извърши операцията.

var circle_radis = 2;
var circle_area = 3.14 * circle_radis * circle_radis;
var result = "Circle area is " + circle_area;
// => Circle area is 12.56

За да може да направи конкатенацията с оператора плюс, javascript трябва да уеднакви типовете на операндите и да превърне числото в низ. 

Това крие и капани:
var a = 5, b = "8";
var result = a + b;
// => 58

Защо се получава 58, а не 13?

Оператори за сравняване на променливи

  • == equal(проверява само стойност)
  • != not equal(проверява само стойност)
  • === identical(проверява стойност и тип)
  • !== not identical (проверява стойност и тип)
  • >, <, >=, <=

Избягвайте == и !=, използвайте === и !== вместо тях:

'' == '0'            // false
0 == ''              // true
0 == '0'             // true

false == 'false'     // false
false == '0'         // true

false == undefined   // false
false == null        // false
null == undefined    // true

' \t\r\n ' == 0   // true

Логически оператори

Логическите оператори позволяват писането на комплексна логика от повече от 1 израз.

  • && - логически и
      true false
    true true false
    false false false
  • || - логическо или
      true false
    true true true
    false true false

Логически оператори

Когато смесвате 2 различни логически оператора трябва да ползвате скоби: 

var a = true, b = false, c = true;
console.log(a && (b || c));

Операндите в логическите оператори могат да бъдат и резултати от функции. 

function screen_is_ready() {
	console.log("screen_is_ready");
	return false;
}
function query_done() {
	console.log("query_done");
	return true;
}
console.log(screen_is_ready() && query_done());

В тази ситуация query_done няма да бъде извикана изобщо.

Оператора !

Оператора ! обръща булевата стойност:

var age = 14;
var is_teenager = age <= 19;
if (!is_teenager) {
	console.log("Allow in");
} else {
	console.log("Go away");
}

Приложен към не-булева променлива, удивителния знак я свежда до булева и обръща стойността й:

var age = 0;
if (!age) {
	console.log("Newborns are not allowed here");
}

Стриктен режим

Strict mode инструктира browser-а да изпълни код в по-стриктна среда и да ви предпази от чести грешки в кода ви.

Вклучва се като се постави низ "use strict" като първи statement в javascript файл, script tag, или във функция. 

<script>
"use strict"
age = 25;
</script>

<script>
"use strict"
var age = 25;
</script>
