Типове от данни

В javascript има 2 вида данни: прости(basic types) и комплексни(complex types). Простите(понякога се наричат и скаларни) се състоят от една стойност. Всички комплексни типове са обекти и могат съдържат повече от 1 стойност.

Днес започваме с разглеждането на простите типове:

  • Numbers(числа)
  • Strings(низове)
  • Booleans(булеви)
  • Undefined и null

Numbers - Числа

Всички числа в javascript се третират като реални; например 5 вътрешно е 5.0.

Функции свързани с числа има в обекта Math: abs, ceil, max, min, random и други.

Деление на нула в javascript дава резултат безкрайност:

5 / 0 => Infinity

Ако операцията от числа не може да бъде извършена се получава стойност NaN(not a number):

5 / "Banana" => NaN

Numbers - operators

Оператори:

  • Събиране 5 + 6 = 11
  • Изваждане 5 - 6 = -1
  • Умножение 5 * 6 = 30
  • Деление 5 / 6 = 0.833...
  • Остатък от деление 5 % 6 = 1
  • Инкрементиране a++, a += b, ++a
  • Декрементиране a--, a -= b, --a
  • Обръщане на знак: -a => a *= -1
  • Форсиране на тип число: +a => a

Strings - Низове

String(низ) в javascript е текст обграден в двойни или единични кавички.

Промяната на низ създава нов низ. Няма отделен тип character; той е низ с един елемент.

Обратната наклонена черта е escape character и се използва, за да се поставят специални символи в низа:

\n, \r, \t - whitespace

\", \' - quoted кавички

Низа не може да съдържа нов ред директно, трябва да се използва \n, \r или и двете

Concatenation("слепване") на 2 низа се прави с оператора плюс:

var name = "Emo";
var greeting = "Good evening";
console.log(greeting + ", " + name);
// => Good evening, Emo

Strings

var text = "Lorem ipsum dolor sit amet, consectetur adipisicing\r\n" + 
"elit. Corporis, ratione, accusantium fugit consectetur pariatur";

console.log("Message for " + text);

Strings - Низове (методи)

Всеки низ е обект и притежава properties и methods:

var s = "javascript strings";
console.log(s.length); 
// => 18

Методите са функции, които работят в контекста на низа. Те не променят низа, а връщат нов низ. Няколко примера:

  • s.indexOf()
  • s.lastIndexOf()
  • s.replace()
  • s.charAt()
  • s.slice()
  • s.substr()
  • s.toUpperCase(), .toLowerCase()

Booleans - булеви променливи

Booleans(булев тип) е тип съдържащ само 2 стойности: истина(true) и лъжа(false).

Всички стойности в останалите типове могат да се сведат до булева стойност.

Следните стойности се свеждат до false(falsy) са:

  • false
  • null, undefined
  • ''(empty string)
  • 0, NaN

Всичко останало се свежда до true(truthy).

Определяне на тип на променлива

Javascript има механизъм за определяне на типа на променлива -- typeof оператора. 

var test;
console.log(typeof test); // undefined
test = 5;
console.log(typeof test); // number
test = new Number(5);
console.log(typeof test); // object

typeof оператора сам по себе си не може да се използва във всички ситуации. Понякога трябва да се прави по-дълбока проверка за типа на променливата.

Най-често той се използва за да се провери дали дадена променлива съществува: 

if (typeof variable === "undefined") {
	console.log("variable doesn't have value yet. ");
} else {
	console.log("variable is " + variable);
}

Функции

Функциите са широка тема, която ще обсъдим в дълбочина по-късно. За момента само ще видим какъв е основния им синтаксис и какво представляват.

Функциите в javascript са обекти, т.е. функция може да присъства навсякъде където може да присъства стойност.

Дефинирането на функция в действителност създава променлива, която се казва както името на функцията и сочи към обекта на функцията. 

function add(a, b) {
	return a + b;
}
// Работи идентично на: 
var add = function (a, b) {
	return a + b;
}

Тъй като функциите са просто обекти, дефинирането на 2 функции с едно и също име просто ще презапише първата стойност с втората.

Аналогично, ако презапишем променливата, сочеща към функцията с нещо различно от функция, ще имаме проблеми.

Анатомия на функция

var add = function (a, b) {
    return a + b;
}
  • var add = Инициализираме променлива, която ще сочи към фунцкията.
  • function Ключова дума за създаване на функция.
  • a, b Списък с аргументи, които са локални променливи във функцията. Промени по аргументите ще се отразят на стойности от complex типове, а тези от basic типовете няма да бъдат променени.
  • { return a + b } Тяло на функцията, което ще бъде изпълнено в момента в който функцията бъде извикана.

Дефинирането на функция само по себе си не извиква кода, който се съдържа в нея. Тя трябва да бъде извикана.

Извикване на функция

Извикването на функция може да се направи по 4 различни начина, но за момента ще разгледаме само най-простия от тях: 

var res = add(4, 5);
console.log(res);

Синтаксиса е да се напише името на променливата, сочеща към функцията с допълнение на кръгли скоби, в които да се подадат аргументите.

В момента, в който се стигне до извикване на функция, javascript спира изпълнението на текущия контекст и предава изпълнението към тялото на функцията.

Изпълнението на кода на функция се прекратява при достигане на конструкция return или при достигане на } за затварянето на функция.

Undefined and Null

В javascript има 2 вида празни променливи:

  • undefined
  • null

Техническата разлика между тях е тънка:

  • undefined: до момента променливата не е била споменавана или е била декларирана без стойност
  • null е стойност, която се дава на променлива от програмиста, за да укаже, че променливата няма стойност

Type Conversion: промяна на тип

Когато оператор трябва да работи с различни типове, javascript вътрешно прави обръщане на типовете, за да извърши операцията.

var circle_radis = 2;
var circle_area = 3.14 * circle_radis * circle_radis;
var result = "Circle area is " + circle_area;
// => Circle area is 12.56

За да може да направи конкатенацията с оператора плюс, javascript трябва да уеднакви типовете на операндите и да превърне числото в низ.

Това крие и капани:

var a = 5, b = "8";
var result = a + b;
// => 58

Защо се получава 58, а не 13?

Оператори за сравняване

  • == equal(проверява само стойност)
  • != not equal(проверява само стойност)
  • === identical(проверява стойност и тип)
  • !== not identical (проверява стойност и тип)
  • >, <, >=, <=

Не ползвайте == и !=, а само === и !==:

'' == '0'            // false
0 == ''              // true
0 == '0'             // true

false == 'false'     // false
false == '0'         // true

false == undefined   // false
false == null        // false
null == undefined    // true

' \t\r\n ' == 0   // true

Логически оператори

Логическите оператори позволяват писането на комплексна логика от повече от 1 израз.

  • && - логически И
      true false
    true true false
    false false false
  • || - логическо ИЛИ
      true false
    true true true
    false true false

Логически оператори

Когато смесвате 2 различни логически оператора трябва да ползвате скоби: 

var a = true, b = false, c = true;
console.log(a && (b || c));

Операндите в логическите оператори могат да бъдат и резултати от функции. 

function screen_is_ready() {
	console.log("screen_is_ready");
	return false;
}
function query_done() {
	console.log("query_done");
	return true;
}
console.log(screen_is_ready() && query_done());

В тази ситуация query_done няма да бъде извикана изобщо.

Оператора !

Оператора ! обръща булевата стойност:

var age = 14;
var is_teenager = age <= 19;
if (!is_teenager) {
	console.log("Allow in");
} else {
	console.log("Go away");
}

Приложен към не-булева променлива, удивителния знак я свежда до булева и обръща стойността й:

var age = 0;
if (!age) {
	console.log("Newborns are not allowed here");
}

Стриктен режим

Strict mode инструктира browser-а да изпълни кода ви в по-стриктна среда, като по този начин ви предпазва от често срещани грешки.

Вклучва се като се постави низ "use strict" като първи statement в javascript файл, script tag, или във функция. 

<script>
"use strict"
age = 25;
</script>

<script>
"use strict"
var age = 25;
</script>
